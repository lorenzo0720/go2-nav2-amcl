# Unitree Go2 SLAM Toolbox — 功能实现与测试记录

本文档汇总在 **Ubuntu 22.04 + ROS 2 Humble + 网线连接 Go2** 上已跑通的功能、环境要求、关键操作与工程修改，便于实验室同学**克隆后复现**。

---

## 一、已实现 / 已验证功能

### 1. 通信与开发环境

| 项目 | 说明 |
|------|------|
| 系统 | Ubuntu 22.04 |
| ROS | ROS 2 **Humble** |
| 宇树接口 | 克隆并编译官方 [unitree_ros2](https://github.com/unitreerobotics/unitree_ros2) 的 `cyclonedds_ws`（消息包 `unitree_go`、`unitree_api`） |
| 网络 | 有线连接 Go2，PC 侧典型网段 `192.168.123.x`；使用 **Cyclone DDS**，并在环境脚本中指定接狗的 **网卡名**（例如 `enp47s0`） |
| Python | 编译 ROS/宇树包前建议 **`conda deactivate`**，避免 Conda 的 `python3` 缺少 `empy` 导致 `unitree_*` 编译失败 |

### 2. SLAM 建图

- 启动：`ros2 launch go2_core go2_start.launch.py`（含驱动、点云→激光、`slam_toolbox`、EKF、RViz 等）。
- 遥控：`ros2 run teleop_twist_keyboard teleop_twist_keyboard`（需 `ros-humble-teleop-twist-keyboard`）。
- 地图保存：在 **`/map` 存在**且建图进程运行时，使用 `nav2_map_server` 的 `map_saver_cli` 保存为 `.yaml` + `.pgm`；保存目录需存在（例如 `mkdir -p ~/go2_maps`）。

### 3. 自主导航（Nav2）

- 启动：`ros2 launch go2_navigation2 go2_nav2.launch.py`  
  **默认地图**：包内 **`go2_navigation2/maps/my_room.yaml`**（与 `.pgm` 一并随仓库安装到 `share/go2_navigation2/maps/`）。  
  自定义：`map:=/完整路径/你的图.yaml`。
- 流程：RViz 中 **2D Pose Estimate** → **Nav2 Goal**。
- 依赖：`ros-humble-navigation2`、`ros-humble-nav2-bringup`、`ros-humble-nav2-map-server`、`ros-humble-xacro` 等（见根目录 `README.md`）。

### 4. 工程内关键修改（相对早期上游）

| 位置 | 内容 |
|------|------|
| `go2_navigation2/launch/go2_nav2.launch.py` | 补全 `twist_bridge`、`footprint_to_link`、`lowstate_to_imu`；`driver` 带 `driver.yaml`；默认地图改为**包内 `maps/my_room.yaml`**。 |
| `go2_navigation2/config/nav2_params.yaml` | `velocity_smoother.feedback: OPEN_LOOP`；`max_speed_xy`、机器人半径与膨胀、`BaseObstacle.scale` 等导航与避障相关调参。 |
| `go2_driver/src/driver.cpp` | 定时发布 `odom`→`base_footprint`，避免无 `robot_pose` 时缺少 `odom` 帧。 |
| `go2_navigation2/maps/` | 示例栅格地图 **`my_room.yaml` / `my_room.pgm`**，供开箱测试；见该目录下 `README.md`。 |

## 二、学弟复现步骤（推荐顺序）

1. **安装** ROS 2 Humble、按官方文档配置 **unitree_ros2** 与 **CycloneDDS + 网卡**（`setup_go2.sh` 一类脚本需按本机修改网卡名）。  
2. **克隆** 本仓库到某目录（例如 `~/unitree-go2-slam-toolbox-main`），**不要**提交 `build/`、`install/`、`log/`（仓库已含 `.gitignore`）。  
3. 打开终端：  
   `conda deactivate`（若用 Conda）  
   `source ~/unitree_ros2/setup_go2.sh`（或你的等价脚本）  
   `cd <工作空间根目录>`  
   `colcon build`  
   `source install/setup.bash`  
4. 验证：`ros2 pkg list | grep go2_core`，能列出则环境正确。  
5. 建图 / 导航命令见根目录 `README.md`。

**常见错误**：只 `cd` 到源码目录、未执行 `source install/setup.bash` → 报 `Package 'go2_core' not found`。

## 三、环境与启动命令备忘

### 1. 每次终端先加载环境（顺序不要颠倒）

```bash
conda deactivate 2>/dev/null || true
source ~/unitree_ros2/setup_go2.sh
# 将下面路径改为你的克隆目录（含 install/ 的工作空间根目录）
source ~/unitree-go2-slam-toolbox-main/install/setup.bash
```

**说明**：第二行路径需与克隆位置一致；**必须先 `colcon build`**。请将 `setup_go2.sh` 中网卡名改为本机**接网线**的接口名。

### 2. 建图

```bash
ros2 launch go2_core go2_start.launch.py
# 另一终端：teleop_twist_keyboard
```

### 3. 保存自建地图（可选）

```bash
mkdir -p ~/go2_maps
ros2 run nav2_map_server map_saver_cli -f ~/go2_maps/my_map
```

### 4. 导航（默认使用仓库内示例地图）

```bash
ros2 launch go2_navigation2 go2_nav2.launch.py
# 指定其他地图：
# ros2 launch go2_navigation2 go2_nav2.launch.py map:=/完整路径/xxx.yaml
```

## 四、测试中出现过的现象与处理要点

| 现象 | 原因 / 处理 |
|------|----------------|
| 编译 `unitree_*` 报 `No module named 'em'` | `conda deactivate` 或 `pip install empy`；系统侧 `python3-empy`。 |
| 缺少 `xacro`、`teleop_twist_keyboard` | `sudo apt install ros-humble-xacro ros-humble-teleop-twist-keyboard`。 |
| 导航只动半步就停 | `velocity_smoother` 使用 **OPEN_LOOP**（见配置说明）。 |
| RViz `Client interrupted while waiting for service` | Nav2 未完全就绪或操作中断；可忽略或待栈完全就绪后再点目标。 |
| 参照物少时图上与真机偏差大 | AMCL 漂移；**2D Pose Estimate** 重定位或降速。 |
| `odom` frame does not exist | 保证 DDS 与 `robot_pose`；`driver` 已定时发 `odom` 占位。 |
